Utilidad

¿Por qué?

Pregunta: ¿No acabo de entender que uso tiene crear el componente desde TypeScript en lugar de por ejemplo añadirlo al template con un ngIf y crearlo y destruirlo dependiendo de la condición?

Respuesta: El típico ejemplo es la creación de formularios de contenido y estructura variable, que vienen dados por un servidor. El servidor te puede pasar todas las preguntas, el tipo de input, etc, y para estos casos lo más apropiado es crearlos de forma dinámica. Intentar gestionar eso desde el template sería, como mínimo, complejo.


Sacar provecho de componentes dinámicos

Respuesta: Para elementos de UI que se usan de forma puntual y que son reutilitzables a los largo de toda la app (alerts, vistas modales, etc), así como en elementos que el usuario puede añadir/eliminar dinámicamente (por ejemplo en un editor de contenido, donde el usuario puede "abrir"nuevos objetos y para cada uno se crea un nuevo tab, o una tabla donde el usuario puede añadir columnas...)

Utiliza componentes dinámicos siempre que tu aplicación pueda o no mostrar cierto componente. Por ejemplo como comenta Enrique, en alertas, en diálogos, en modales y también por ejemplo en menús de acciones el típico lateral para crear/editar.
