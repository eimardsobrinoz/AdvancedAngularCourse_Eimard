Using Resolve

Datos cargados a través del servicio route y su Resolver que llama un servicio que trae los datos antes de cargar el componente

Es útil cuando tienes una lista de elementos y accedes a uno de ellos a través del enrutado pasando como parámetro su id para ver sus detalles en el componente de detalle

y en este componente recoges el parámetro y haces una llamada a servicio para obtener los datos. De este modo se cargaría los datos después del componente en sí

Mediante la esta clase mycustomResolver que implementa el servicio Resolve, te permite acceder a la información de enrutado de tipo ActivateRouterSnapshot entre otros y hacer la llamada a servicio con ese id y obtener los datos, de modo que podrían estar disponible estos desde el componente, antes de que este se cargue.

Se establece una key para almacenar estos datos en this.route.snapshot.data['nombre-de-mi-key']

Si nos fijamos, en ese mismo nivel se guarda los datos que se meten en el atributo data a la hora de navegar.
